# 工单管理

查看和处理用户提交的工单,支持回复和关闭工单。

## 表格列说明

### 标题 (Title)
工单标题。

### 用户 (User)
提交工单的用户。

### 状态 (Status)
工单当前状态(带颜色指示器):
- **待跟进** (红色) - 新提交,等待管理员处理
- **待回复** (黄色) - 管理员已回复,等待用户
- **已解决** (绿色) - 问题已解决
- **已关闭** (灰色) - 工单已关闭

### 更新时间 (Updated At)
最后更新时间。

## 表格操作

### 回复 (Reply)
打开工单对话抽屉:
- 查看工单完整对话
- 输入回复内容
- 发送回复后状态变为"待回复"

### 关闭 (Close)
关闭工单(需确认):
- 状态变为"已关闭"
- 关闭后无法再操作
- 用户可以再次查看

### 查看 (Check)
查看已关闭工单的对话记录(只读)。

## 工单对话

打开工单后显示对话界面:

**对话消息**:
- 工单描述(用户首条消息)
- 所有跟进记录
- 显示时间戳
- 区分用户/管理员消息

**回复工单**:
1. 在底部输入框输入回复
2. 点击发送按钮
3. 消息立即显示
4. 状态自动更新为"待回复"

**自动刷新**:
- 每5秒自动刷新对话
- 获取最新消息

## 筛选功能

### 状态筛选
选择"已关闭"查看关闭的工单。

## 使用场景

### 场景一: 处理新工单

1. 查看"待跟进"状态工单
2. 点击"回复"按钮
3. 阅读用户问题
4. 输入解决方案
5. 发送回复
6. 用户收到通知

### 场景二: 解决问题后关闭

1. 打开工单对话
2. 确认问题已解决
3. 点击"关闭"按钮
4. 确认关闭操作
5. 工单标记为已关闭

### 场景三: 多轮对话

1. 打开工单
2. 查看历史对话
3. 回复新消息
4. 持续跟进直到解决

## 重要提示

1. **及时回复**: 待跟进工单应尽快处理
2. **对话记录**: 所有消息永久保存
3. **自动刷新**: 对话每5秒自动更新
4. **关闭限制**: 关闭后无法再回复
5. **状态变化**: 回复后自动变为"待回复"
6. **用户通知**: 回复后用户会收到通知
7. **查看已关闭**: 已关闭工单可以查看但不能回复
